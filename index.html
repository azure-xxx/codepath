単なるエラーメッセージの翻訳ではない。「なぜそのコードを書いたか」まで推定し、根本的な誤解を解く。

😰 こんなエラーが出たとき

const nums = [1, 2, 3]; const doubled = nums.map(n * 2); console.log(doubled);
TypeError: nums.map is not a function
at line 2, col 21

🤖 AIの3層解析

🔴 根本原因

map() は関数を受け取りますが、n * 2 という式を渡しています。カッコの中に直接計算式は書けません。

💡 なぜそう書いた？（推定）

「mapに何をするかを書けばいい」という理解は正しい。ただし「する処理」は関数の形で渡す必要があります。n => n * 2 がその書き方です。

✅ 正しいコード

nums.map(n => n * 2) — アロー関数で包むことで、nを受け取って2倍にする関数になります。
